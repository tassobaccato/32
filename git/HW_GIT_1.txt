TXT
 1. Создать внешний репозиторий c названием TXT:
	на странице repositories кликнуть New
	в поле Repository name: TXT
	✓ Add a README file
	Create repository

 2. Клонировать репозиторий TXT на локальный компьютер:
	git clone https://github.com/tassobaccato/TXT

 3. Внутри локального TXT создать файл “new.txt”:
	touch TXT/new.txt

 4. Добавить файл под гит:
	cd TXT
	git add new.txt

 5. Закоммитить файл:
	git commit -m "added file"

 6. Отправить файл на внешний GitHub репозиторий:
	git push

 7. Отредактировать содержание файла “new.txt” - написать информацию о себе (ФИО, возраст, 
количество домашних животных, будущая желаемая зарплата). Всё написать в формате TXT:
	vim new.txt
	ФИО: Арбабаева Касила Кушербаевна,
        Возраст: 18+,
       	Количество домашних животных: null,
        Будущая желаемая зарплата: от 100000

 8. Отправить изменения на внешний репозиторий:
	git add . && git commit -m "added change" && git push

 9. Создать файл preferences.txt
	cat > preferences.txt

 10. В файл preferences.txt” добавить информацию о своих предпочтениях (Любимый фильм, любимый 
сериал, любимая еда, любимое время года, сторона которую хотели бы посетить) в формате TXT:
	Любимый фильм: "Белое солнце пустыни",
        любимый сериал: null,
        любимая еда: итальянская кухня,
        любимое время года: весна,
        сторона которую хотели бы посетить: Ирландия

 11. Создать файл skills.txt добавить информацию о скиллах которые будут изучены на курсе в формате TXT:
	Документация тестирования SDLC, STLC.
	Клиент-серверная архитектура.
	HTTP методы запросов, коды ответов, структуры HTTP запросов и ответов.
	JSON, XML. Их структура.
	Тестирование API через Postman (JS, автотесты API).
	Снятие и чтение логов c внешнего сервера.
	Снифинг http web трафика через Charles и Fiddler.
	Dev Tools веб браузеров (Google Chrome, FireFox).
	VPN. (Как работает, зачем нужен, как использовать, варианты инструментов).
	Мобильное тестирование.
	Особенность iOS, Android, гайдлайны.
	Сборка iOS приложений на XCode.
	Сборка Android приложений на Android Studio.
	ADB (управление андройд девайсами).
	Настройка прокси и vpn на iOS и Android.
	Перехват (сниффинг) мобильного трафика через Charles и Fiddler на iOS и Android.
	Командная строка (terminal) Linux.
	Основы bash скриптинг, автоматизация рутинных задач на сервере.
	Доступ к удалённым серверам.
	Основы SQL.
	База данных Postgres.
	Нереляционная база данных Redis.
	Нагрузочное тестирование в Jmeter.
	Методология разработки Scrum.
	Python. (Основы. Создание клиент серверного приложения).

 12. Сделать коммит в одну строку:
	git add . && git commit -m "added files"

 13. Отправить сразу 2 файла на внешний репозиторий:
	git push

 14. На веб интерфейсе создать файл bug_report.txt:
	add file -> create new file

 15. Сделать Commit changes (сохранить) изменения на веб интерфейсе:
	commit new file

 16. На веб интерфейсе модифицировать файл bug_report.txt, добавить баг репорт в формате TXT:
	edit this file
	id:  1
	title: кнопка "каталог" не кликабельна
	path: 1.открыть главную страницу
	    2.кликнуть кнопку "каталог"
	expected: открывается меню каталога
	actual: меню не открывается
	priority: 1
	comment: img

	id: 2
	title: на карточке товара нет цены
	path: 1.в поле поиска ввести артикль 123
	expected: на карточке отображается цена
	actual: на карточке отсутствует цена
	priority: 2
	comment: img

 17. Сделать Commit changes (сохранить) изменения на веб интерфейсе.
 18. Синхронизировать внешний и локальный репозиторий TXT:
	git pull
